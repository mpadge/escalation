# Stage 003 — Centrality–Escalation Correlation

Files in reading order:

- **plan.md** — Stage design plan: question (does escalation concentrate network power?), GP emulation approach for ε–degree correlation surfaces, and open questions.
- **tasks.md** — Concrete implementation checklist; includes refactoring of shared utility scripts and the full Stage 3 pipeline.
- **results/gp.md** — GP emulator training results for the C_lo and C_hi correlation surfaces: ARD hyperparameters, RMSE, coverage, and baseline correlation statistics.
- **results/summary.md** — Quantitative phase diagram findings: C_hi − C_lo diff ranges per parameter pair, which pairs show positive vs. negative diff, and the dissociation result.
- **results/interpretation.md** — Prose interpretation: where escalation concentrates individual power, what structural parameter governs that advantage, and the dissociation between population-level amplification and individual centrality concentration.
- **design-decisions.md** — Key design decisions made during implementation: shared utility file naming, difference surface normalisation, column auto-detection, and integration with Stage 2.
- **.transcript.md** — Semi-anonymised session transcript; records design discussion and rationale behind decisions.
