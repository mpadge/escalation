# Session Transcript: bivariate-sensitivity

## Session Overview

Stage 006 extended the sensitivity analysis pipeline to the bivariate (ε, σ) model
introduced in Stage 005, running Morris OAT screening and Sobol decomposition for
the new estimand `psi_sigma` (sensitivity of ε̄(∞) to μ_σ perturbation) and
producing a recoverability check confirming partial reproduction of Stage 002/003
rankings under degenerate σ conditions.

## Design Decisions Made

### Decision 1: Triple-paired run via run_sigma_paired
**Chosen:** A new `run_sigma_paired` function runs three simulations per seed —
(a) mu0_lo / nominal mu_sigma, (b) mu0_hi / nominal mu_sigma, (c) mu0_lo /
mu_sigma + 0.1 — computing both `psi` and `psi_sigma` from a single binary call.
**Rationale:** Reusing the same seed for all three sims cancels stochastic variance
from both estimates simultaneously. The third sim costs one extra forward run per
design point; the R analysis pipeline then reads both columns from the same output CSV.
**Tradeoffs:** Each Morris/Sobol run is now 50% more expensive than the univariate
case (3 sims vs 2 per design point). The delta_mu_sigma=0.1 was chosen as 10% of
the mu_sigma range [0.5, 2.0]; smaller values would amplify numerical noise.
**Proposed by:** joint

### Decision 2: eta_obs held fixed in bivariate Morris
**Chosen:** `eta_obs` was moved from `param_names` to `fixed` (at its reference
value pars_a$mid_eta_obs = 0.05) throughout Morris and Sobol, reducing the bivariate
free-parameter set from 15 to 14.
**Rationale:** Both eta_obs and mu_sigma scale the observational-learning pathway;
simultaneous variation would create a near-collinear direction in the design space
that the Morris elementary-effects estimator cannot resolve cleanly.
**Tradeoffs:** The joint sensitivity of eta_obs × mu_sigma is left uncharacterised;
deferred to Stage 007.
**Proposed by:** mpadge

### Decision 3: sigma_decay=0.002 in degenerate recoverability run
**Chosen:** recover-bivar.R fixes sigma_decay at its default (0.002) rather than 0.0.
**Rationale:** The task specification listed sigma_decay=0.002 as the degenerate
fixed value. With non-zero decay, sigma drifts downward from mu_sigma=1.0 during
the run, producing a profile that differs from the strict original model (which had
no sigma dynamics at all).
**Tradeoffs:** Resulted in Spearman ρ = 0.78 rather than the ≥ 0.95 threshold.
Setting sigma_decay=0.0 would give exact degenerate recovery (confirmed by T005-8)
and likely achieve ρ ≥ 0.95. This is an identified fix for Stage 007 or a re-run.
**Proposed by:** mpadge (spec); agent (identified the gap post-run)

### Decision 4: #[serde(default)] on Params struct
**Chosen:** Added `#[serde(default)]` at the struct level so design CSVs produced
by old analysis scripts (without sigma columns) deserialise without error.
**Rationale:** The Stage 005 Params extension broke backward-compatible CSV reading
for the existing morris and sobol make targets. The struct-level default attribute
is the minimal fix without requiring format migration.
**Tradeoffs:** If a design CSV accidentally omits a sigma column, the field silently
defaults rather than erroring; acceptable because design files are always
machine-generated by the R scripts.
**Proposed by:** agent

## Tradeoffs Considered

- **sigma_decay=0.0 vs 0.002 in recover-bivar**: Using 0.0 would exactly replicate
  T005-8 conditions and guarantee exact degenerate recovery, but the task spec listed
  0.002. The post-run ρ=0.78 makes the tradeoff visible; fix is straightforward.
- **15 vs 14 free params in Morris**: Including eta_obs would give a complete 15-param
  screen but at the cost of collinearity artefacts in the Morris EE estimates.

## Open Questions

- Whether sigma_decay=0.0 in recover-bivar.R lifts Spearman ρ above 0.95 (expected yes).
- GP emulation for psi_sigma is deferred to Stage 007; Stage 006 Morris/Sobol identified
  mu_sigma and sigma_sigma as dominant (by μ*) but Sobol shows strong interactions (S1≈0,
  ST≈0.86–1.03), suggesting the GP surface may be non-smooth.
- Whether the eta_obs × mu_sigma interaction warrants a joint-variation screen in Stage 007.
