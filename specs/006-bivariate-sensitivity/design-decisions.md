# Design Decisions: Stage 006 — Bivariate Sensitivity Analysis

## Summary

Stage 006 ran Morris OAT screening and Sobol variance decomposition for the bivariate
(ε, σ) model, computing both the new `psi_sigma` estimand (sensitivity of ε̄(∞) to
μ_σ perturbation) and `psi` (μ₀ sensitivity with σ active) from a single binary
invocation. The primary finding is that psi_sigma variance is driven almost entirely
by parameter interactions (S1 ≈ 0, ST ≈ 0.86–1.03), with mu_sigma and sigma_sigma
leading by μ* but no single parameter dominant in isolation.

## New Design Decisions

### Decision 1: Triple-paired run infrastructure (run_sigma_paired)
**Chosen:** `run_sigma_paired` in `src/experiment.rs` runs three simulations per
seed per design point: (a) mu0=0.4 / nominal mu_sigma, (b) mu0=0.6 / nominal
mu_sigma, (c) mu0=0.4 / mu_sigma + 0.1. Both `psi` and `psi_sigma` are computed
from a single binary invocation and written to the same output CSV.
**Rationale:** Shared seed cancels stochastic variance from both estimates; one
extra forward run per design point is the minimum cost for the second estimand.
**Tradeoffs:** 50% more expensive per design point than the univariate pipeline.
delta_mu_sigma = 0.1 (10% of the [0.5, 2.0] range) is the finite-difference step;
it was validated by the T006-3 test (psi_sigma ≈ 0 in degenerate case, all 5 seeds).
**Proposed by:** joint
**Relates to:** Stage 000 paired-run design; `run_paired` in experiment.rs.

### Decision 2: eta_obs fixed at reference value in bivariate Morris/Sobol
**Chosen:** `eta_obs` is excluded from `param_names` and fixed at 0.05 (its
reference midpoint) throughout Morris-bivar and Sobol-bivar. The 14-parameter
bivariate candidate set is: original 11 minus eta_obs, plus mu_sigma, sigma_sigma,
eta_sigma, sigma_decay.
**Rationale:** eta_obs and mu_sigma both scale the observational-learning pathway
(eta_obs globally, mu_sigma per-agent). Simultaneous variation creates near-collinear
elementary-effect directions; Morris cannot resolve them reliably.
**Tradeoffs:** The joint eta_obs × mu_sigma sensitivity is uncharacterised until
Stage 007. The effective candidate set is 14, not 15, free parameters.
**Proposed by:** mpadge

### Decision 3: #[serde(default)] on Params struct
**Chosen:** `#[serde(default)]` added at the struct level in `src/params.rs`, making
all Params fields optional in CSV deserialisation (defaulting to Rust Default values
when absent).
**Rationale:** Stage 005 added four new Params fields (mu_sigma, sigma_sigma,
eta_sigma, sigma_decay) that are absent from design CSVs generated by the pre-Stage-005
analysis scripts. Without `serde(default)`, the existing `make morris` and `make sobol`
targets would fail to read their own design files.
**Tradeoffs:** Silent defaulting if a design CSV accidentally omits a field. Acceptable
because design CSVs are always machine-generated by the R scripts.
**Proposed by:** agent

### Decision 4: sigma_decay=0.002 in degenerate recoverability run (known gap)
**Chosen:** recover-bivar.R fixes sigma_decay at 0.002 (the structural default), not
0.0, for the degenerate σ comparison.
**Rationale:** The task specification listed sigma_decay=0.002. With non-zero decay,
sigma drifts downward from mu_sigma=1.0 during the run, producing a trajectory that
is not exactly the original model (which had no σ dynamics).
**Tradeoffs:** Spearman ρ = 0.78 between Stage 002/003 archived μ* rankings and the
degenerate Stage 006 rankings — below the ≥ 0.95 threshold. The top-3 parameters
match (alpha 1→2, gamma 2→1, lambda 3→3); mid-range diverges (eta_obs drops 4→8,
beta 5→7). Setting sigma_decay=0.0 in the fixed list would give exact degenerate
recovery (per T005-8) and is the recommended fix for a follow-on run.
**Proposed by:** mpadge (spec); agent (identified post-run)

## Integration with Prior Work

The triple-paired run infrastructure extends the Stage 000 `run_paired` / `run_experiment`
pattern with a third simulation per seed. `cmd_sensitivity` in `main.rs` now calls
`run_sigma_paired` for both Morris and Sobol subcommands, so all future sensitivity
runs emit `psi_sigma` alongside `psi` without any R-side changes.

The Morris finding that `alpha` re-leads for the `psi` metric (μ* = 0.636, rank 1)
with σ active is consistent with the Stage 003 result that influence locality governs
μ₀ amplification. Activating σ does not displace alpha from the top of the Ψ ranking.

For `psi_sigma`, mu_sigma (μ* = 0.490) and sigma_sigma (μ* = 0.359) lead the σ-trait
parameters, but network structure parameters (lambda, dw_obs, dw_bridge, alpha) all
contribute substantially — the σ amplification mechanism is not self-contained.

## Issues Resolved

- **psi_sigma never populated**: Resolved by `run_sigma_paired`; the `psi_sigma:
  Option<f64>` field in `RunSummary` that was always `None` since Stage 005 is now
  populated for every Morris and Sobol design point.
- **CSV backward compatibility**: Resolved by `#[serde(default)]` on Params; old
  design files deserialise cleanly with sigma fields defaulting to their Rust defaults.

## Deferred Items

- **sigma_decay=0.0 recoverability re-run**: straightforward fix to recover-bivar.R;
  expected to lift Spearman ρ from 0.78 to ≥ 0.95.
- **GP emulation for psi_sigma** (Stage 007): Morris + Sobol identified top parameters
  but S1 ≈ 0 / high ST suggests strong interactions; the GP surface may require more
  design points or a non-stationary kernel.
- **eta_obs × mu_sigma joint variation** (Stage 007): deferred from Stage 006 to avoid
  collinearity in the Morris design.
- **Split σ_emit / σ_recv** (from Stage 005): still deferred pending GP results on
  the unified σ pathway.

## Process Notes

- The `make progress` compatibility question was raised before the first long run;
  confirmed that `run_sigma_paired` writes one `.done` file per design row (1 seed),
  matching the expected count from `wc -l design_csv - 1`.
- The Sobol result (all S1 ≈ 0, all ST ≈ 0.86–1.03) was unexpected given that
  Morris identified clear μ* separation across parameters. The implication is that
  psi_sigma is driven by parameter interactions rather than main effects — a qualitatively
  different structure from the univariate Ψ and a meaningful finding for Stage 007 planning.
- The recoverability shortfall (ρ = 0.78) was diagnosed post-run as a sigma_decay
  artefact rather than a structural failure of the degenerate recovery property;
  T005-8 and T006-3 both confirm exact degenerate recovery under sigma_decay=0.0.
