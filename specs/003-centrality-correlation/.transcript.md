# Session Transcript: Stage 3 — Centrality–Escalation Correlation

## Session Overview

Stage 3 extended the two-GP emulation pipeline from Stage 2 to a new response
variable: the Pearson correlation between agent escalation propensity (ε) and
weighted in-degree at quasi-equilibrium. The session covered shared utility
extraction, refactoring of Stage 2 scripts, writing Stage 3 training/phase/plot
scripts, running the full pipeline, and producing quantitative results and
interpretive write-up.

## Design Decisions Made

### Decision 1: Shared utility files without stage-number suffixes
**Chosen:** `gp_train_utils.R`, `gp_phase_utils.R`, `plot_utils.R` (no integer
suffix)
**Rationale:** Stage-numbered utility files (`gp_train_utils3.R`) would
proliferate with each new stage. A single shared file per utility class is
sourced by all stage scripts.
**Tradeoffs:** Any breaking change to a utility signature requires updating all
callers across stages; accepted as worth the deduplication benefit.
**Proposed by:** mpadge

### Decision 2: Generic `response_col` and `derived_col` parameters
**Chosen:** `build_design_matrix(raw, response_col)` and
`write_phase_csvs(..., derived_col)` rather than hardcoding response names.
**Rationale:** Allows Stage 2 to pass `"mean_epsilon_final"` and Stage 3 to
pass `"epsilon_k_corr_final"` without separate functions.
**Tradeoffs:** Callers must know the column name; type errors surface at
runtime rather than load time.
**Proposed by:** agent

### Decision 3: Column-name auto-detection in plot utilities
**Chosen:** `build_e_limits` detects the value column by excluding known
coordinate columns; `plot2.R` detects `e_val_col` from the first CSV file.
**Rationale:** DiceKriging was unavailable during development, so the Stage 2
phase CSVs on disk used the old `"psi"` column name while the refactored script
writes `"val"`. Auto-detection avoids breaking `make plots2` against stale CSVs.
**Tradeoffs:** Fragile if a CSV gains an unexpected extra numeric column;
acceptable given the controlled file format.
**Proposed by:** agent

### Decision 4: Difference surface on raw correlation scale (not normalised)
**Chosen:** `diff = C_hi − C_lo` directly, not divided by 0.2 as Ψ was in
Stage 2.
**Rationale:** Both C_lo and C_hi are already on the same bounded [−1, 1]
correlation scale; normalisation by the μ₀ step size (0.2) would make
interpretation less intuitive.
**Tradeoffs:** Diff values are not directly comparable to Ψ.
**Proposed by:** agent

### Decision 5: T003-3 references output CSV rather than console output
**Chosen:** Task description updated to read `results/gp3_hyperparams.csv`
and `results/gp3_data.csv` rather than capturing transient console output.
**Rationale:** `save_hyperparams_csv` already writes all the required numbers;
pointing to the file is more reproducible.
**Tradeoffs:** None significant.
**Proposed by:** mpadge

## Tradeoffs Considered

- **Stage-numbered vs. shared utilities**: stage-numbered files are safer
  (no cross-stage breakage risk) but create duplication that grows with each
  stage; shared files win on maintainability.
- **Diff normalised vs. raw**: normalised diff would mirror the Ψ construction
  from Stage 2 but is less interpretable for a bounded correlation variable;
  raw diff chosen.
