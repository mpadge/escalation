# Session Transcript: Stage 007 — GP Bivariate Emulation

## Session Overview

Stage 007 completed GP training and phase diagram generation for the bivariate (ε, σ) model. A runtime error in the train/test stratification was fixed before phase grids and plots were produced across three axis pairs.

## Design Decisions Made

### Decision 1: ntile() for stratified train/test split
**Chosen:** Replace `cut(quantile(..., probs=seq(0,1,0.2)))` with `dplyr::ntile(..., 5L)` in `split_train_test_bivar`.
**Rationale:** The `psi_sigma_mean` distribution contains exact zeros (design points where sigma perturbation has no effect), causing `quantile()` to return duplicate breakpoints that `cut.default` rejects. `ntile()` assigns ranks and partitions them into equal-sized bins, never producing duplicate boundaries regardless of ties.
**Tradeoffs:** `ntile()` splits on rank rather than value, so strata boundaries are not at even quantile values; this is acceptable for a train/test balance heuristic.
**Proposed by:** agent

### Decision 2: Load pre-trained GP models in gp_phase_bivar.R
**Chosen:** `gp_phase_bivar.R` reads `gp_bivar_psi_sigma.rds` and `gp_bivar_psi.rds` saved by `gp_train_bivar.R` rather than re-fitting the GPs.
**Rationale:** The fitted GP objects are already serialised by the training script. Re-fitting in the phase script would double the DiceKriging O(n³) cost for no gain; the existing pipeline (`gp_phase.R` loading `gp_psi.rds`) follows this same pattern.
**Tradeoffs:** The phase script is now coupled to the training script's output filenames; changing the `.rds` naming requires updating both files.
**Proposed by:** agent

### Decision 3: Archived Ψ=1 overlay skipped with warning
**Chosen:** The Stage 003 archived phase CSVs in `results/003-centrality-correlation/gp_phase/` do not include lambda as an axis and have no alpha×lambda pair. The overlay is skipped with a `cli_alert_warning` for both the alpha and lambda axis pairs.
**Rationale:** Stage 003 used alpha, gamma, beta, eta_obs as its top four parameters; lambda was not among them. There is no archived surface that spans both mu_sigma × alpha and a lambda cross-section simultaneously. Erroring would prevent plot generation; warning preserves all other output.
**Tradeoffs:** The cross-stage Ψ=1 boundary comparison (a stated goal in the plan) is deferred. A Stage 008 item could archive a dedicated univariate alpha × lambda phase CSV.
**Proposed by:** joint

### Decision 4: Shared diverging colour scale across both panels
**Chosen:** Both the `psi_sigma` and `psi` panels within each plot share a symmetric diverging scale (`[-abs_max, abs_max]`) computed from the union of both estimands' value ranges.
**Rationale:** A shared scale makes the two panels directly visually comparable; the diverging midpoint at 0 means both panels use the same zero-amplification reference.
**Tradeoffs:** If one estimand has a much wider range, the other panel's colour variation is compressed. Given both estimands are dimensionless sensitivity ratios of similar magnitude this is acceptable.
**Proposed by:** agent

## Tradeoffs Considered

- **Re-fit GPs in phase script vs. load .rds**: loading wins on cost and consistency with prior pipeline.
- **Error vs. warn for missing archived overlay**: warn wins to preserve the three output PNGs even when archival data is unavailable.
