# Session Transcript: Stage 005 — Add Status Sensitivity

## Session Overview

The session extended the model from a univariate agent trait (escalation propensity ε)
to a bivariate pair (ε, σ), where σ is status sensitivity — the per-agent multiplier
on observational learning and prestige radiation. The full design was settled through
iterative clarification before implementation began.

## Design Decisions Made

### Decision 1: Status sensitivity (σ) as the second dimension
**Chosen:** σ_i ∈ [0,1] governing observational learning strength and prestige radiation
magnitude, rather than bond/bridge orientation (β) or reciprocity orientation (ρ).
**Rationale:** σ directly gates the observational cascade by which a μ₀ perturbation
propagates beyond direct participants. It controls the conductivity of the social
environment to escalation signals, making it the strongest lever for the primary
estimand Ψ(θ, μ_σ). β and ρ were judged to act more indirectly on network topology.
**Tradeoffs:** σ is conceptually closer to ε than β or ρ, risking endogenous correlation;
this was accepted as a feature (emergent ε–σ correlation is itself an output of interest).
**Proposed by:** joint

### Decision 2: Global parameters retained as scales; σ multiplies them
**Chosen:** eta_obs and dw_obs are kept as global reference scales. σ_i multiplies both:
prestige radiation becomes `dw_obs · σ_w · σ_k` and observer propensity update becomes
`eta_obs · σ_k`. Setting mu_sigma=1.0, sigma_sigma=0.0 recovers the original model exactly.
**Rationale:** An earlier proposal to retire eta_obs/dw_obs entirely (Option B) was rejected
because it would break backward compatibility and produce a collinearity problem in
sensitivity analysis. Retaining the globals as scales gives mu_sigma=1.0 as a clean
degenerate baseline.
**Tradeoffs:** Globals and μ_σ cannot be varied independently in sensitivity analysis
without care; accepted as a known limitation.
**Proposed by:** mpadge (identified the backward-compatibility gap); agent (proposed the fix)

### Decision 3: Unified σ (no emit/receive split)
**Chosen:** A single σ_i applies to both emission (how strongly a winner projects status)
and reception (how strongly an observer updates). The product σ_w · σ_k means status
effects require both a projecting winner and a receptive observer.
**Rationale:** Simplicity; splitting into σ_emit and σ_recv would add a second parameter
without clear empirical motivation at this stage.
**Tradeoffs:** Low-σ populations produce quadratically suppressed status effects (σ² not σ).
**Proposed by:** mpadge

### Decision 4: σ evolves via vicarious reinforcement; sigma_decay prevents saturation
**Chosen:** Observer σ_k updates by sign(winner_payoff_delta) · eta_sigma. Global sigma_decay
(default 0.002) is applied each timestep to prevent σ saturating at 1.0.
**Rationale:** Without decay, winners almost always gain payoff, so σ drifts monotonically
upward for all observers. Decay was prescribed in the plan as a contingency if validation
showed collapse; T005-9 confirmed the collapse, and decay was added.
**Tradeoffs:** sigma_decay is an additional parameter; deferred to the follow-on sensitivity stage.
**Proposed by:** agent (decay design); mpadge (confirmed vicarious reinforcement signal)

### Decision 5: Stage scope limited to code + validation
**Chosen:** Stage 005 covers only model extension and Phase 1–3 validation tests.
The sensitivity analysis pipeline (Morris → Sobol → GP) is deferred to a subsequent stage.
**Rationale:** The σ update rule added complexity; cleaner to validate the model extension
before scaling up the analysis.
**Proposed by:** mpadge

## Tradeoffs Considered

- **Option B (retire globals) vs Option 1 (keep globals as scales)**: Option B was initially
  preferred for cleanliness, but Option 1 was adopted after the backward-compatibility gap
  was identified — mu_sigma=0 under Option B produces zero observational effects, not the
  original model behaviour.
- **Winner payoff delta vs observer's own payoff change**: Observer payoff never changes
  within a single timestep (non-participants), so the winner's payoff gain was used as a
  proxy vicarious signal. Noted as a design simplification.

## Open Questions

- Whether sigma_decay should be coupled to eta (like kappa = eta_obs / eta) or treated
  as independent in the sensitivity analysis.
- Whether the σ_w · σ_k product produces excessively weak effects at low μ_σ; a sqrt
  or linear formulation may be needed.
- Full sensitivity analysis (Morris → Sobol → GP) for the bivariate (ε, σ) model is
  deferred to stage 006.
